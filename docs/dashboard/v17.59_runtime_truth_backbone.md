# Claire v17.59 — Runtime Truth Backbone + Dashboard Data Contract

## Purpose

v17.59 establishes the canonical runtime truth layer the redesigned dashboard needs before additional UI expansion.

The dashboard must read real runtime truth for:

- run state
- route state
- terminal state
- stage execution
- validation result
- evidence chain
- memory eligibility
- runtime health

## Canonical output

```text
exports/latest/dashboard_runtime_truth.json
```

## Status output

```text
src/frontend/command_center/modern/runtime_truth_status.json
```

## No fake data rule

The builder does not fabricate completed work. Missing required stage truth is marked as blocked, pending, skipped_by_route, or not_applicable depending on route context.

## Main command

```bash
python tools/claire_build_runtime_truth.py
```

You can also provide a specific file:

```bash
python tools/claire_build_runtime_truth.py --input exports/latest/core_run_output.json
```
