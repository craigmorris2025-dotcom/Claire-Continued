# Claire v17.80 - Launch Candidate Freeze

This build freezes the current local platform state as a launch candidate.

## Purpose

After endpoint smoke proof, launch hardening, desktop startup reliability, and manual browser/Swagger proof binder creation, Claire needs a freeze layer so cleanup or deletion does not accidentally break the working candidate.

## Routes

- `GET /platform/launch-candidate`
- `GET /platform/launch-candidate/summary`
- `POST /platform/launch-candidate/freeze`
- `GET /platform/freeze/status`

## Output files

- `data/launch_candidate/v17_80_launch_candidate_freeze.json`
- `data/launch_candidate/launch_candidate_manifest.json`
- `data/launch_candidate/protected_paths_manifest.json`
- `data/launch_candidate/v17_80_launch_candidate_stop_go.json`
- `data/launch_candidate/v17_80_launch_candidate_stop_go.md`
- `data/dashboard/launch_candidate_freeze_payload.json`

## Critical rule

Do not delete or archive folders after this freeze without a proof-based cleanup review. The backend folder especially remains review-only until cleanup proof confirms it is safe.
