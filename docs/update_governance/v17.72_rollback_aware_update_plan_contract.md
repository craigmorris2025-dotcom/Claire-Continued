# Claire v17.72 - Rollback-Aware Update Plan Contract

This build creates rollback-aware planning for staged update packs.

## Purpose

Before any update pack can execute, Claire must know how to restore the previous state.

v17.72 creates rollback plans only. It does not run updates and does not run rollback restores.

## API routes

- `GET /updates/rollback-plan`
- `GET /updates/rollback-plan/summary`
- `POST /updates/rollback-plan/create`
- `GET /updates/rollback/status`

## Output files

- `data/update_packs/rollback_plans/<plan_id>.json`
- `data/update_packs/rollback_plan_index.json`
- `data/dashboard/rollback_update_plan_payload.json`

## Critical rule

No update execution without rollback proof. No rollback execution without operator review. No hidden background update behavior.
