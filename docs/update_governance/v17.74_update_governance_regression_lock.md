# Claire v17.74 - Update Governance Regression Lock

This build locks the update governance chain created in v17.71-v17.73.

## Purpose

The lock verifies that update pack staging, rollback planning, automatic runner gate, and internet readiness do not accidentally enable hidden execution.

## API routes

- `GET /updates/regression-lock`
- `GET /updates/regression-lock/summary`
- `POST /updates/regression-lock/rebuild`
- `GET /updates/governance/lock`

## Output files

- `data/update_packs/update_governance_regression_lock.json`
- `data/dashboard/update_governance_regression_lock_payload.json`

## Critical rule

No hidden updates. No background execution. No uncontrolled self-modification.

Automatic updates remain disabled until the full proof pack confirms operator review, rollback, validation, source trust, scheduler locks, and route safety.
