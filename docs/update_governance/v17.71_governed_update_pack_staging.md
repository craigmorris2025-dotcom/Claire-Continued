# Claire v17.71 - Governed Update Pack Staging

This build creates the staging layer for future governed update packs.

## Purpose

Claire can prepare update packs, but this build does not execute them.

Update packs must include:

- pack identity
- source authority
- scope
- file operations
- validation plan
- rollback plan
- operator review
- execution gate

## API routes

- `GET /updates/staging`
- `GET /updates/staging/summary`
- `POST /updates/staging/create`
- `GET /updates/governance/status`

## Output files

- `data/update_packs/staging/<pack_id>.json`
- `data/update_packs/update_pack_staging_index.json`
- `data/dashboard/update_pack_staging_payload.json`

## Critical rule

No hidden updates. No automatic execution. No uncontrolled self-modification.

v17.71 stages update packs only. v17.72-v17.74 must prove rollback, execution gates, validation locks, and operator review before any automatic update path can exist.
