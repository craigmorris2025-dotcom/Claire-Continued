# Claire v17.84.1 - Live Probe Body Compatibility Repair

The uploaded Swagger response showed a 422 validation error expecting `raw_input` while the live probe body used:

```json
{
  "query": "OpenAI latest research",
  "confirm_text": "RUN GOVERNED WEB PROBE"
}
```

This repair adds a compatibility adapter so the governed live probe accepts both `query` and `raw_input`.

## Use this Swagger endpoint

`POST /internet/live-probe/run-confirmed`

Body:

```json
{
  "query": "OpenAI latest research",
  "confirm_text": "RUN GOVERNED WEB PROBE"
}
```

Alternative body:

```json
{
  "raw_input": "OpenAI latest research",
  "confirm_text": "RUN GOVERNED WEB PROBE"
}
```

## Safety

This repair does not enable automatic browsing, automatic updates, or runtime-truth ingestion.
