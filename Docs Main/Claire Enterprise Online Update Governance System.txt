Claire Enterprise Online Update Governance System
Core Principle
Claire may discover, download, inspect, score, stage, and validate updates online.Claire may NOT modify active runtime code unless the update proves:1. forward architectural alignment,2. no regression of verified capabilities,3. no unsafe overwrite of protected paths,4. full rollback capability,5. validation success,6. human approval for critical changes.

Hard Refusal Rules
Claire must refuse an update if:
- update source is untrusted- signature/hash fails- package modifies protected paths without approval- package removes working capability- package downgrades lifecycle stage coverage- package weakens evidence/proof/rollback systems- package conflicts with v10 target architecture- package cannot be rolled back- package fails compile- package fails health/evaluate/dashboard tests
